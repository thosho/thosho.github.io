Engaine

An advanced, privacy-focused Android application designed to run and manage AI models directly on your mobile device, while offering the flexibility to connect to powerful cloud-based AI APIs.

Built using modern Android development practices (Kotlin, Jetpack Compose) and a robust background service architecture.

View on GitHub

Core Features & Capabilities

Dual AI Execution Modes

  • Local On-Device AI: Engaine runs LLMs offline using your phone's processor. Integrates custom-compiled llama.cpp for .gguf models and Google's MediaPipe for .task models.
  • Privacy First: Your data never leaves your device when using local models.
  • Cloud AI API Support: Instantly toggle to Cloud Mode. Supports secure API connections to Google Gemini, Groq, and HuggingFace for state-of-the-art models.

Hardware Diagnostics

  • RAM & Device Safety: Actively monitors your phone's available RAM in real-time.
  • Auto-Scaling: On lower-end devices (e.g., 4GB RAM), it automatically limits the AI's context length and thread count to prevent Android system crashes.
  • Smart Recommendations: Live dashboard diagnostics tell you exactly how much free RAM you have and recommend maximum model sizes.

Isolated "Brain" Architecture

  • AI engines run in a completely isolated background process (:brain).
  • If a massive imported model causes a fatal memory crash, only the background "brain" dies.
  • The main UI stays perfectly alive, catches the crash, and displays a user-friendly error message telling you how to fix it.

RAG (Retrieval-Augmented Gen)

  • Built-in document analysis system to generate embeddings for imported text files.
  • Chat directly with your documents. The AI reads your files and answers questions based specifically on the contained information.

UI & Model Management

  • Glassmorphic Design: Highly modern, dark-mode, translucent "glass" UI built entirely in Jetpack Compose.
  • Model Registry: Built-in file manager to import .gguf or .task files from downloads, organize them, and delete old ones to save storage.
  • Test Playground: Dedicated chat interface to test out loaded models (local or cloud) with real-time response generation.

Detailed Summary

1. Dual AI Execution Modes (Local & Cloud)

Local On-Device AI: Engaine can run Large Language Models (LLMs) completely offline using your phone's processor. It integrates a custom-compiled llama.cpp engine to run modern .gguf models, and Google's MediaPipe engine to run specialized .task models.

Privacy First: Your data never leaves your device when using local models.

Cloud AI API Support: If your phone lacks the RAM for a large model, you can instantly toggle to Cloud Mode. The app supports secure API connections to Google Gemini, Groq, and HuggingFace, allowing you to use state-of-the-art models over the internet.

2. Intelligent Hardware Diagnostics

RAM & Device Safety: Local AI requires significant memory. Engaine actively monitors your phone's available RAM in real-time.

Auto-Scaling: If you are on a lower-end device (like a 4GB RAM phone), the app automatically limits the AI's context length and thread count to prevent the Android system from crashing.

Smart Recommendations: The dashboard provides live diagnostics, telling you exactly how much free RAM you have and recommending the maximum model size your phone can handle safely.

3. Isolated "Brain" Architecture

AI models can sometimes crash if they run out of memory. Engaine solves this by running the AI engine in a completely isolated background process (:brain). If an imported model causes a fatal memory crash, only the background "brain" dies. The main user interface stays perfectly alive, catches the crash, and displays a user-friendly error message telling you how to fix it.

4. RAG (Retrieval-Augmented Generation)

Engaine includes a built-in document analysis system. You can import your own documents or text files, and the app will generate embeddings for them. You can then chat directly with your documents. The AI will read your files and answer questions based specifically on the information contained within them.

5. UI and Model Management

Glassmorphic Design: The app features a highly modern, dark-mode, translucent "glass" user interface built entirely in Jetpack Compose.

Model Registry: A built-in file manager allows you to easily import .gguf or .task files from your phone's downloads folder, organize them, and delete old ones to save storage space.

Test Playground: A dedicated chat interface to test out the capabilities of the currently loaded model (whether local or cloud) with real-time response generation.
